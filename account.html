Account API Documentation

Base URL: http://34.121.64.128:5000/api/account

1. Create Account

Endpoint: /create

Method: POST

Description: Registers a new user by creating an account.

Request Body:

{
  "phone_number": "string",
  "email_address": "string",
  "password": "string",
  "first_name": "string",
  "last_name": "string",
  "gender": "string",
  "dob": "YYYY-MM-DD"
}

Success: 200 OK

{
  "status": true,
  "message": "mail sent"
}

Failure: 201 Conflict 

{
  "success": false,
  "message": "User already exist with similar email/phone number!"
}

2. Verify Account

Endpoint: /verify

Method: POST

Description: Verifies the account registration using a PIN sent via email.

Request Body:

{
  "pin": "number"
}

Success: 200 OK

{
  "status": true,
  "message": "Registered successfully!"
}

Failure: 400 Bad Request 

{
"status": false,
"message": "invalid pin / expired pin"
}

3. Login

Endpoint: /login

Method: POST

Description: Logs in a user and returns a JWT token for authentication.

Request Body:

{
  "email_address": "string",
  "password": "string"
}

Success: 200 OK

{
  "success": true,
  "message": "Login successful",
  "token": "jwt_token",
  "data": {
    "id": "user_id",
    "email": "user_email"
  }
}

Failure: 401 Unauthorized 

{
  "success": false,
  "message": "Account not found" 
}

4. Forget Password

Endpoint: /forget

Method: POST

Description: Initiates the password reset process by sending a PIN to the user's email.

Request Body:

{
  "email_address": "string"
}

Success: 200 OK

{
  "status": true,
  "message": "mail sent"
}

Failure: 401 Unauthorized 

{
  "success": false,
  "message": "Account not found"
}

5. Verify Password Reset

Endpoint: /verify-forget-password

Method: POST

Description: Verifies the PIN received for password reset.

Request Body:

{
  "pin": "number"
}

Success: 200 OK

{
  "status": true,
  "email_address": "user_email",
  "message": "Password reset pin verification is successful"
}

Failure: 400 Bad Request 

{
  "status": false,
  "message": "invalid pin / expired pin"
}

6. Reset Password

Endpoint: /reset-password

Method: POST

Description: Resets the user's password using a new password.

Request Body:

{
  "email_address": "string",
  "new_password": "string"
}

Success: 200 OK

{
  "status": true,
  "message": "Password reset is successful"
}

Failure: 401 Unauthorized 

{
  "success": false,
  "message": "Account not found"
}

7. Change Password

Endpoint: /change-password

Method: POST

Description: Changes the user's current password to a new password.

Request Body:

{
  "id": "user_id",
  "old_password": "string",
  "new_password": "string"
}

Success: 200 OK

{
  "status": true,
  "message": "Password updated successfully! Please login with your new password"
}

Failure: 401 Unauthorized

{
  "status": false,
  "message": "Wrong old password!" 
}

Note:Use the provided token for authenticated requests as necessary.
