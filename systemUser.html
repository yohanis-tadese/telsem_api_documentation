System User API Documentation

Base URL: http://34.121.64.128:5000/api/system-users

1. Create System User

Endpoint: /

Method: POST

Description: Creates a new system user with specified details.

Request Body:


{
  "full_name": "John Doe",
  "email_address": "john.doe@example.com",
  "phone_number": "+1234567890",
  "group_id": 2 // is only valid group id accepted
}

Success: 201 Created 


{
  "success": true,
  "message": "System user created successfully",
  "data": {
    "id": 1,
    "full_name": "John Doe",
    "email_address": "john.doe@example.com",
    "phone_number": "+1234567890",
    "group_id": 2,
    "last_logged_in": null
  }
}

Failure: 400 Bad Request 


{
  "success": false,
  "message": "Email address already in use"
}

Possible Causes:

  • The email address might already be registered.
  • The request body might be missing required fields.

3. Get System User by ID

Endpoint: /:id

Method: GET

Description: Retrieves a system user by its ID.

Response:


{
  "id": 1,
  "full_name": "John Doe",
  "email_address": "john.doe@example.com",
  "phone_number": "+1234567890",
  "group_id": 1,
  "last_logged_in": null
}

Success: 200 OK


{
  "id": 1,
  "full_name": "John Doe",
  "email_address": "john.doe@example.com",
  "phone_number": "+1234567890",
  "group_id": 1,
  "last_logged_in": null
}

Failure: 404 Not Found 


{
  "success": false,
  "message": "System user not found"
}

4. Update System User

Endpoint: /:id

Method: PATCH

Description: Updates details of a system user by ID.

Request Body:


{
  "full_name": "John Doe",
  "email_address": "john.doe@example.com",
  "phone_number": "+1234567890",
  "group_id": 1
}

Success: 200 OK


{
  "success": true,
  "message": "System user updated successfully",
  "data": {
    "id": 1,
    "full_name": "John Doe",
    "email_address": "john.doe@example.com",
    "phone_number": "+1234567890",
    "group_id": 1,
    "last_logged_in": null
  }
}

Failure: 400 Bad Request 


{
  "success": false,
  "message": "Invalid system user ID"
}

5. Delete System User

Endpoint: /:id

Method: DELETE

Description: Deletes a system user by ID.

Success: 204 No Content 


{
  "success": true,
  "message": "System user deleted successfully"
}

Failure: 404 Not Found 


{
  "success": false,
  "message": "System user not found"
}

6. Login

Endpoint: /login

Method: POST

Description: Authenticates a system user and returns a token.

Request Body:


{
  "email_address": "john.doe@example.com",
  "password": "myStrongP@ssword"
}

Success: 200 OK


{
  "success": true,
  "message": "Login successful",
  "data": {
    "userId": 1,
    "email": "john.doe@example.com",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "groupId": 1,
    "roles": [
      {
        "canView": true,
        "canViewDetail": true,
        "canAdd": false,
        "canUpdate": true,
        "canDelete": false,
