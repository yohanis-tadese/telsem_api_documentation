System User Group API Documentation

Base URL: http://34.121.64.128:5000/api/system_user_groups

1. Create System User Group

Endpoint: /

Method: POST

Description: Creates a new system user group with specified details.

Request Body:


{
  "name": "Admin Group",
  "description": "Group with administrative access to all features",
  "roles": [
    {
      "canView": true,
      "canViewDetail": true,
      "canAdd": true,
      "canUpdate": true,
      "canDelete": true,
      "page": "Dashboard",
      "title": "Dashboard"
    },
    {
      "canView": true,
      "canViewDetail": true,
      "canAdd": true,
      "canUpdate": true,
      "canDelete": true,
      "page": "Users",
      "title": "User Management"
    },
    {
      "canView": true,
      "canViewDetail": true,
      "canAdd": true,
      "canUpdate": true,
      "canDelete": true,
      "page": "Settings",
      "title": "System Settings"
    }
  ]
}

Success: 201 Created 


{
  "status": true,
  "message": "Admin group created successfully!",
  "data": {
    "id": 1,
    "name": "Admin Group",
    "description": "Group with administrative access to all features",
    "roles": [
      {
        "canView": true,
        "canViewDetail": true,
        "canAdd": true,
        "canUpdate": true,
        "canDelete": true,
        "page": "Dashboard",
        "title": "Dashboard"
      },
      {
        "canView": true,
        "canViewDetail": true,
        "canAdd": true,
        "canUpdate": true,
        "canDelete": true,
        "page": "Users",
        "title": "User Management"
      },
      {
        "canView": true,
        "canViewDetail": true,
        "canAdd": true,
        "canUpdate": true,
        "canDelete": true,
        "page": "Settings",
        "title": "System Settings"
      }
    ]
  }
}

Failure: 400 Bad Request 


{
  "status": false,
  "message": "Invalid input data!"
}

Possible Causes:

  • The request body may be missing required fields.
  • The provided data may not meet validation criteria.
  • Validation & Length:
    • name: Required, max length 20 characters.
    • description: Optional, max length 50 characters.
    • roles: Required, should be a JSON array.
